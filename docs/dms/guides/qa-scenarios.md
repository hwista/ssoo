# DMS QA 시나리오

> 최종 업데이트: 2026-03-23

표준화 리팩토링 이후 회귀 확인에 사용하는 최소 QA 시나리오입니다.

---

## 1. 문서 CRUD

- 기존 문서를 열고 본문을 수정한 뒤 저장한다.
- 기존 문서의 메타데이터를 수정한 뒤 저장한다.
- 기존 문서를 삭제하고 탭/목록 상태를 확인한다.
- 새 문서를 빈 문서로 생성하고 저장한다.
- 새 문서를 템플릿 기반으로 생성하고 저장한다.
- 새 문서를 AI 요약 기반으로 생성하고 저장한다.
- 저장 시 파일 이름 또는 경로를 바꾸고 이동 결과를 확인한다.

## 2. AI Compose

- 인라인 AI 작성을 실행하고 본문이 편집기에 반영되는지 확인한다.
- 제목/경로 추천이 sidecar 또는 저장 흐름에 반영되는지 확인한다.
- 참조 파일 첨부 후 관련성 경고가 적절히 표시되는지 확인한다.
- compose 실패 후 재시도가 가능한지 확인한다.
- compose 중 중단 또는 오류 후 화면 상태가 복구되는지 확인한다.

## 3. References

- 참조 파일을 추가하고 sidecar 상태가 동기화되는지 확인한다.
- 참조 파일을 삭제하고 soft-delete 상태가 반영되는지 확인한다.
- 삭제한 참조 파일을 복원한다.
- 템플릿을 선택하고 적용 상태를 확인한다.
- 템플릿을 해제하고 복원한다.
- 복원 실패 항목에 대해 재시도를 수행한다.
- 첨부파일을 업로드하고 저장 시 mutation 이 반영되는지 확인한다.

## 4. Editor / Navigation

- undo/redo 가 기대한 순서로 동작하는지 확인한다.
- diff 모드를 열고 다시 편집 모드로 돌아온다.
- `Ctrl+S` 로 저장이 동작하는지 확인한다.
- TOC 클릭 시 해당 본문 위치로 이동하는지 확인한다.
- 본문 링크 클릭 시 viewer/editor 스크롤과 하이라이트가 동작하는지 확인한다.
- 이미지 클릭 시 미리보기 뷰어가 열리는지 확인한다.

## 5. Settings / Search

- 설정값을 수정하고 저장한다.
- 템플릿을 생성, 수정, 삭제한다.
- AI 검색을 실행하고 결과 목록이 렌더링되는지 확인한다.
- 검색 결과를 열고 문서 탐색 흐름이 이어지는지 확인한다.

---

## 기록 방식

- 실패 시나리오는 재현 조건과 화면/route 맥락을 함께 남깁니다.
- DMS 고유 회귀인지, 전역 환경 문제인지 구분해서 기록합니다.
- 표준화 트랙 마감 이후에도 기능 고도화 작업의 기본 회귀 체크리스트로 재사용합니다.
