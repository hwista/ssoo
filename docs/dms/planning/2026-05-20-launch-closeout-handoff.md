## 3. 이번 작업에서 구현/정리된 내용

### 3.1 AI 요약과 검색 결과

- 원문 열람 권한이 없는 문서라도 검색 결과에 포함된 경우 AI 요약은 표시되도록 정리했습니다.
- 서버 Docker 런타임에 AI provider 환경이 반영되지 않아 요약이 0건이던 문제를 환경 반영과 재빌드로 해결했습니다.
- 일반 OpenAI provider/package/compose 임의 변경은 롤백했고, 기존 Azure OpenAI runtime 설정을 유지했습니다.

### 3.2 AI 검색 사이드카와 검색 기록

- AI 검색 사이드카의 불필요한 전체 제목을 제거했습니다.
- 검색 기록 저장소를 브라우저 저장소에서 DB 기준으로 전환했습니다.
- “내 자주 검색”, “인기 검색어”, “검색 기록”을 동시에 표시합니다.
- 인기 검색어는 다음 조건을 만족해야만 노출됩니다.
  - 검증/테스트 검색어 저장 차단
  - popular 집계에서도 검증/테스트 패턴 제외
  - 전체 검색 횟수 2회 이상
  - 서로 다른 사용자 2명 이상
- 현재 실제 popular가 비어 있을 수 있는데, 기준 충족 데이터가 없으면 정상입니다.

### 3.3 잠긴 문서 미리보기

- 권한 없는 문서를 클릭하면 즉시 권한 요청 팝업을 띄우지 않고 문서 탭을 엽니다.
- 문서 화면은 일반 편집기가 아니라 잠긴 문서 전용 화면으로 렌더링됩니다.
- 서버가 허용한 상단 일부 미리보기만 표시합니다.
- 이미지, 첨부, 표 성격의 라인은 미리보기에서 제외하도록 필터링했습니다.
- 나머지 원문은 클라이언트로 내려주지 않는 방향을 확인했습니다.
- 권한 요청 CTA를 누르면 기존 권한 요청 다이얼로그가 열립니다.
- 마크다운이 아닌 권한 없는 파일은 미리보기를 만들지 않고 기존처럼 차단합니다.

## 6. 바로 이어서 할 다음 작업

### 1순위: 권한 없는 검색 결과 카드 스니펫/키워드 노출 정리

목표:

- 권한 없는 문서는 검색 결과에서 “발견 가능” 상태와 AI 요약은 유지하되, 원문 기반 스니펫/키워드가 과하게 노출되지 않게 정리합니다.

권장 구현 방향:

1. 서버 검색 응답에서 권한 없는 결과의 원문 스니펫/키워드 필드를 제거하거나 preview-only 정책으로 제한합니다.
2. 웹 결과 카드는 권한 없는 결과에 대해 “문서 화면에서 제한된 미리보기를 확인하고 권한을 요청하세요” 톤으로 표시합니다.
3. AI 요약은 기존 요구대로 유지합니다.
4. 브라우저에서 검색 결과 카드에 원문 민감 문구가 노출되지 않는지 확인합니다.

검증 게이트:

```bash
pnpm run build:server
pnpm run build:web-dms
pnpm run verify:access-dms:raw
pnpm run codex:preflight
docker compose up -d --build server dms
```

### 2순위: 변경사항 커밋 전 diff 정리

현재 작업트리에는 Copilot이 진행하던 기존 변경과 이번 세션 변경이 섞여 있습니다. 커밋 전에는 의도하지 않은 변경이 들어가지 않았는지 전체 diff 리뷰가 필요합니다.

### 3순위: Prisma migration 산출물 확인

DB 기반 검색 기록 모델이 추가되었으므로, 실제 배포 기준에서는 정식 migration 파일 필요 여부를 확인해야 합니다.
