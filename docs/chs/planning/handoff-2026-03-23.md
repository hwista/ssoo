# CHS 핸드오프 - 2026-03-23

> 작성일: 2026-03-23
> 목적: CHS 신규 프로젝트 작업을 다음 세션에서 바로 이어갈 수 있도록 현재 상태와 우선순위를 정리

---

## 현재 상태 요약

- CHS 앱(`apps/web/chs`)은 Next.js 15 기반으로 부팅 가능하며, 피드/게시판 목록/게시판 상세/게시물 상세/프로필/검색/설정/알림 UI까지 연결됐습니다.
- 서버 측 CHS 모듈(`apps/server/src/modules/chs`)은 `board`, `comment`, `feed`, `follow`, `notification`, `post`, `profile`, `skill`까지 구현돼 있습니다.
- 클라이언트는 Axios 엔드포인트와 TanStack Query 훅 기반으로 게시판 상세, 게시물 상세, 검색, 댓글 트리, 프로필 설정, 팔로우/스킬/경력 관리, 알림 드롭다운까지 실제 동작 가능한 수준으로 올라왔습니다.
- 문서 측면에서는 `docs/chs` 루트만 있었고 planning 문서가 비어 있었으나, 이번에 진입점과 backlog, 본 핸드오프 문서를 추가했습니다.

---

## 이번 점검에서 확인한 사실

### 구현됨

- 홈 피드는 [`apps/web/chs/src/components/pages/feed/FeedPage.tsx`](../../../../apps/web/chs/src/components/pages/feed/FeedPage.tsx) 에서 `useFeed()`로 실제 피드 데이터를 조회합니다.
- 글 작성은 [`apps/web/chs/src/components/pages/feed/ComposeBox.tsx`](../../../../apps/web/chs/src/components/pages/feed/ComposeBox.tsx) 에서 `useCreatePost()`를 사용합니다.
- 게시판 목록은 [`apps/web/chs/src/components/pages/board/BoardListPage.tsx`](../../../../apps/web/chs/src/components/pages/board/BoardListPage.tsx) 에서 `useBoards()`로 조회하고 상세 경로로 이동합니다.
- 게시판 상세는 [`apps/web/chs/src/components/pages/board/BoardDetailPage.tsx`](../../../../apps/web/chs/src/components/pages/board/BoardDetailPage.tsx) 에서 게시판 메타와 게시물 목록을 함께 표시합니다.
- 게시물 상세는 [`apps/web/chs/src/components/pages/post/PostDetailPage.tsx`](../../../../apps/web/chs/src/components/pages/post/PostDetailPage.tsx) 와 [`apps/web/chs/src/app/(main)/post/[id]/page.tsx`](../../../../apps/web/chs/src/app/(main)/post/[id]/page.tsx) 에서 고유 라우트와 댓글 기본 펼침을 제공합니다.
- 검색 화면은 [`apps/web/chs/src/components/pages/search/SearchPage.tsx`](../../../../apps/web/chs/src/components/pages/search/SearchPage.tsx) 에서 키워드/스킬 필터/페이지네이션을 처리합니다.
- 댓글 UI는 [`apps/web/chs/src/components/pages/feed/CommentSection.tsx`](../../../../apps/web/chs/src/components/pages/feed/CommentSection.tsx) 와 하위 컴포넌트에서 트리 렌더링과 답글 작성을 지원합니다.
- 프로필은 [`apps/web/chs/src/components/pages/profile/ProfilePage.tsx`](../../../../apps/web/chs/src/components/pages/profile/ProfilePage.tsx) 에서 내 프로필/타 사용자 프로필 조회, 팔로우/언팔로우, 팔로워·팔로잉 목록, 스킬/경력 관리를 처리합니다.
- 설정 화면은 [`apps/web/chs/src/components/pages/settings/SettingsPage.tsx`](../../../../apps/web/chs/src/components/pages/settings/SettingsPage.tsx) 에서 `profilesApi.update()` 계약에 맞는 프로필 편집을 저장합니다.
- 헤더 알림은 [`apps/web/chs/src/components/layout/NotificationDropdown.tsx`](../../../../apps/web/chs/src/components/layout/NotificationDropdown.tsx) 에서 unread count 및 읽음 처리를 제공하고, `post` 참조를 게시물 상세 라우트로 연결합니다.
- 서버는 `COMMENT`, `FOLLOW`, `REACTION` 이벤트에서 `createNotification()`을 호출해 실제 알림 레코드를 생성합니다.
- 게시물 카드는 수정/삭제 메뉴와 수정 다이얼로그를 제공하고, 게시물 상세는 댓글 섹션 스크롤을 지원합니다.

### 남은 확인/후속

- 다계정 기준 수동 검증으로 팔로우, 스킬 추천, 게시물 수정/삭제, COMMENT/FOLLOW/REACTION 알림 유입을 재확인하면 운영 신뢰도를 높일 수 있습니다.
- 이미지 업로드와 게시판 생성은 서버 엔드포인트/관리 정책이 준비되면 후속 범위로 다시 열 수 있습니다.

### 운영/규칙 이슈

- `pnpm run codex:preflight` 실행 시 2026-03-23 기준 `.github/scripts/check-docs.js` 가 `docs/chs` 도메인을 몰라 실패했습니다.
- 이를 해결하기 위해 `check-docs`에 `chs` 도메인을 허용하고, Codex 미러 규칙 파일 [`/.codex/instructions/chs.instructions.md`] 는 새로 추가했습니다.
