# CRM 문서

> 작성: 2026-06-08

이 폴더는 외부 영업관리 데모를 SSOO 프레임워크와 디자인 문법에 맞게 이식하기 위한 CRM 정본 문서입니다.

## 현재 상태

- 현재 추적 파일 기준 SSOO 저장소에는 CRM 웹 앱(`apps/web/crm`), 서버 모듈(`apps/server/src/modules/crm`), 공유 타입(`packages/types/src/crm`)의 1차 골격이 있습니다.
- CRM 웹은 3001 포트 기준으로 compose에 등록되어 있으며, 현재 화면은 SSOO 업무 화면으로 확정된 구현이 아니라 외부 데모를 이식하기 전의 샘플 조회 기준선입니다.
- CRM 서버 API는 `/api/crm/opportunities` 목록/상세와 검색/상태 필터/정렬 쿼리를 제공하지만, 현재 데이터는 DB 원장이 아니라 `opportunity.fixture.ts`의 5건 샘플입니다. 화면에서도 샘플 모드와 실 DB/저장/확정 미연결 상태를 명시합니다.
- 사용자가 제공한 외부 데모 소스와 DB 스키마, 영업관리 소개 자료를 분석해 1차 PRD와 백로그를 만들었습니다.
- PMS 병렬 기준선과 맞춰 CRM은 계약/청구/매출/원가의 원장성 편집을 맡고, PMS는 실행 수행과 읽기용 인계 스냅샷만 소비하는 경계로 고정했습니다.
- 계정, 권한, 법인, 조직, 관리자 운영은 CRM 내부가 아니라 공용 Admin/공용 권한 모델로 관리하는 전제를 반영했습니다.
- 보호된 발표자료 1개는 본문 추출이 되지 않아 현재 기준선에는 미반영했습니다.

## 즉시 실행 기준

1. CRM DB 1차 모델은 아직 샘플 API 이후 과제로 남긴다.
2. fixture 기반 값은 운영 KPI처럼 보이지 않게 샘플 모드로 명확히 표시한다.
3. 첫 구현은 영업기회 목록과 선택 상세/등록/수정 입력 기준에 집중하되, PMS와 같은 실제 업무 목록 패턴(loading/error, pagination, row selection, detail/open behavior)을 후속으로 맞춘다.
4. 견적, 계약, DMS, PMS 연결은 실제 구현 전까지 미구현으로 표시한다.
5. SSOO 공용 셸과 디자인 토큰을 우선 적용하되, 내부 경계 설명을 제품 화면의 주 콘텐츠처럼 과도하게 노출하지 않는다.
